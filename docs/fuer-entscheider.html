KI-Tempo — mit einer Qualitätsgarantie an der Schnittstelle

CDD gibt der KI-Entwicklungsschleife ein Gate, das so streng und unabhängig ist wie eine gute CI-Pipeline. Erst wenn es grün ist, gilt „fertig" — gemessen, nicht behauptet.

Das Problem in einem Satz

Teams lassen KI in Schleifen laufen, die sich selbst nachbessern — aber „fertig" heißt heute oft „der Agent sagt es selbst" oder „die KI-eigenen Tests sind grün". Das ist ein selbst-genehmigter Pull Request: schnell, aber ohne unabhängige Prüfung. Folgen: ungewisse Qualität, Blackbox-Wissen, unkontrollierte Kosten.

Unsere Antwort

Ein getyptes Modell der Anforderung (prüfbar, geteiltes Wissen) + ein fälschungssicheres Orakel: die KI, die Code schreibt, vergibt nicht selbst „grün". Ein gefälschter grüner Test wird beweisbar abgewiesen. Open Source, CI-grün, reproduzierbar.

Einstieg

Der Geschäftswert

  • Tempo + Qualität zugleich — KI-Geschwindigkeit mit Garantie an der Schnittstelle.
  • Weniger Blackbox-Risiko — das „Warum" bleibt als Spec im Team.
  • Kostenkontrolle — gerichteter Loop statt Brute-Force-Token-Verbrauch (Trade-off, kein Selbstläufer).
  • Günstigeres Review — Konvergenz gegen die Spec prüfen statt jede Zeile.

Ehrlich, der Stand

Kein „Coding is solved". Trägt für formalisierbare Kriterien — nicht für UX/Geschmack/Architektur. Der Mensch setzt und verantwortet die Anforderung. Forschungsprototyp, zwei kleine Fallstudien. Diese Ehrlichkeit ist Absicht.

Der Ask

Ein kleiner, abgegrenzter Pilot: eine echte, formalisierbare Anforderung aus dem Backlog · ~1–2 Wochen · ein Entwickler + ich · Ergebnis: gemessene Konvergenz + Token-/Zeit-Telemetrie als Entscheidungsgrundlage. Risiko klein, Erkenntnis konkret.

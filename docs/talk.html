„My job is to write loops."

Boris Cherny (Anthropic, Claude Code) — „Loop Engineering"

Ein Generator erzeugt, ein Schritt prüft, das Ergebnis fließt zurück — bis zu einem Abbruchkriterium.

Die Architektur ist korrekt. Die offene Frage ist eine andere.

Das Problem ist nicht der Loop.

Es ist sein Terminierungs-Orakel.

Woran erkennt der Loop, dass er fertig ist?

Terminiert auf „Agent sagt fertig" oder Exit-0 …

… ist die Abbruchbedingung an keine externe, prüfbare Referenz gebunden. Was bleibt:

  • Brute Force — Token-Verbrauch ohne verbürgten Endzustand
  • Blackbox — Korrektheit erst nachträglich per Diff-Lektüre
  • Intent Debt (Storey 2026) — „the absence of externalized rationale that developers and AI agents need to work safely with code"

Das „Warum" hat dann u. U. zu keinem Zeitpunkt in einem Kopf existiert.

Die Leitfrage

Wo liegt die Grenze, an der Verifikation auf eine gesetzte Invariante angewiesen ist —

und lässt sie sich typisieren?

Intent-Formalisierung ist eine benannte Grand Challenge (Lahiri 2026). Wir beanspruchen keine Idee — wir liefern eine lauffähige Instanz.

Die Methode: Konvergenz gegen Spec

Getypte Referenz

SPOT — ein F#-DU-Graph, ein git-versioniertes JSON pro Knoten. Konvergenz: Aligned / Pending / Diverged / Orphaned.

Generator ⊥ Orakel

Generator: kein Schreibrecht auf den Code-Pfad (Allowlist). Orakel SetzeSpecAligned: Aligned bei abgedecktem Test-Marker, Greenness via CI-Suite — Test-PASS, nicht Exit-0.

Review = Konvergenz gegen die Spec, nicht der gelesene Diff.

Der gefangene Mis-Spec

Requirement ändert sich: Gebühren. Naiver Entwurf zieht sie beim Sender ab, schreibt sie nirgends gut — Wert verschwindet.


Invariante  spec-werterhaltung : Σ aller Konten = konstant

  $ dotnet test
  ✗ Falsifiable, after 4 tests       ← ohne dass ein Diff gelesen wurde

Fix = Doppik (die Gebühr ist Gegenbuchung). Geld verschwindet nicht.

Das stärkste Bild: das Gate beim Scheitern


$ ./demo-gate-at-failure.sh
  1) korrektes Modell            6/6  grün
  2) naives Modell injiziert     (Gebühr verschwindet)
  3) Orakel:  Falsifiable        rot   ← verwirft das falsche Modell
  4) revert                      6/6  grün

Ein grüner Lauf ist ein Cursor-Klon mit Vokabular. Ein erzwungenes rotes Resultat ist der Beweis.

Drei Schichten an zwei Invarianten

  • Typen — int64-Cent: eine Fehlerklasse kann syntaktisch nicht entstehen
  • FsCheck-Property — über Zufallsfolgen geprüft (fand den Mis-Spec)
  • Lean-4-Beweis — für jede Folge bewiesen

theorem werterhaltung (hb : Hauptbuch) (bs : List Buchung) :
    total (buchen hb bs) = total hb

Ehrlich: zwei Invarianten sind Lean-bewiesen (Werterhaltung + Nichtnegativität); die übrigen (Isolation, Replay) sind FsCheck-only.

Der Logik-gegen-LLM-Hebel: A/B

Jede tragende Behauptung deklariert sich:

A — maschinell belegt

Commit-Hash · arXiv-ID · CI-Run · sorry-Audit. Re-runbar. Immun gegen LLM-Halluzination.

B — menschliche Setzung

„Trifft die Spec die Welt", wer die Invariante setzt, Neuheit, Skalierung. Ehrlich markiert.

Keine unmarkierte Assertion. Die Last-tragenden Aussagen ruhen auf Artefakten, nicht auf meinem Wort.

Reflexiv: das System gatet sich selbst

Ein Test prüft eine Invariante über das eigene .spot/-Modell: kein Aligned-Test-Knoten ohne Test-Marker im Code — mit grüner CI-Suite folgt: kein Aligned ohne grünen Test.

Das System härtet seinen eigenen Drift —

und macht sichtbar, dass der äußerste Bezugspunkt eine menschliche Setzung bleibt.

Ehrliche Grenzen

  • N = 2 Fallstudien — Anwendbarkeit, nicht Skalierung
  • Zwei Lean-bewiesene Invarianten; weitere Roadmap
  • Verifizierer-akzeptiert ≠ korrekt (VeriAct) — das Orakel allein genügt nicht
  • Der Mensch bleibt am Modell-Gate — operational und normativ

Das Setzen der Grund-Invariante ist im System nicht selbst-fundierbar (Regress, kein Unmöglichkeitsbeweis). Wir wählen den Menschen, weil Intent extern und verantwortungsbehaftet ist — nicht, weil eine Maschine es nicht könnte. (Penrose/Lucas abgelehnt.)

Eine Schraube weiter

Das Problem ist nicht der Loop, sondern sein Orakel.

Tausche „Agent sagt fertig" gegen „Konvergenz gegen eine getypte Spec" — und der Loop wird diszipliniert. Der Rest wird nicht eliminiert, sondern typisiert: an die Stelle gelegt, an der ein Mensch ihn setzt — sichtbar, versioniert, gegen Drift gegatet.
